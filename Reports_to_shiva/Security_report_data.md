## 7. Recommended Fixes

### Immediate (do today)

```bash
# 1. Install and enable firewall
sudo apt install ufw -y
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh          # if SSH is used
sudo ufw allow from 192.168.0.0/16 to any port 18789  # LAN-only for gateway
sudo ufw enable

# 2. Disable GNOME Remote Desktop (if not actively needed)
systemctl --user disable gnome-remote-desktop
systemctl --user stop gnome-remote-desktop

# 3. Fix Flask servers to bind to localhost only
# In alexa_skill.py and voice_chat.py, change:
#   app.run(host='0.0.0.0', port=...)
# to:
#   app.run(host='127.0.0.1', port=...)

# 4. Kill the duplicate Cloudflare tunnel on port 5001 (ngrok already handles it)
kill 2424831

# 5. Change openclaw-gateway bind address to 127.0.0.1
# (config location TBD — Shiva to check OpenClaw gateway config)
```

### Short-term

- **Move API keys to environment variables** — never hardcode in scripts
- Create a `.env` file in `~/workspace/scripts/` with restricted permissions (`chmod 600`)
- Update scripts to use `os.getenv('OPENCLAW_TOKEN')` etc.
- **Rotate the exposed keys** — both the OpenClaw bearer token and ElevenLabs key in `voice_chat.py` have been in a Cloudflare-tunneled public server since Mar 6

### Long-term

- Set up **WireGuard VPN** for Shiva's remote access instead of RustDesk
- Use **Fail2Ban** to block brute-force attempts on any remaining open ports
- Add **basic HTTP auth** (or proper token auth) to Flask endpoints
